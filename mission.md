content = """# Mission — Chrome Web Store Success

## Objective

Build high-quality Chrome extensions that achieve:
- Strong product-market fit
- High engagement and retention
- Consistent user growth
- Monetisation potential

The goal is not to build “cool tools” — the goal is to build **products people use daily**.

---

## 🧠 Core Philosophy

- Build for **real demand**, not ideas
- Simplicity beats feature overload
- Distribution matters as much as product
- Speed of iteration > perfection

Every extension should solve a **clear, painful, repeat problem**.

---

## 🎯 Success Criteria

An extension is successful if it:

- Gets installed consistently (growth)
- Gets opened frequently (retention)
- Solves a repeatable use-case (habit)
- Has clear upgrade or monetisation paths

---

## 🚀 Product Principles

### 1. Market First

Before building:
- Identify what users already want
- Validate demand (search trends, Reddit, X, App Store reviews)

---

### 2. Clear Value Proposition

Each extension must be explainable in one line:

> “This helps you do X faster/easier/better.”

If it’s not obvious in 3 seconds, it will not convert.

---

### 3. First Impression Wins

Chrome Web Store success depends heavily on:
- Name
- Icon
- Screenshots
- First 5 seconds of use

Design for **instant clarity**.

---

### 4. Daily Use > Occasional Use

Prioritise:
- tools people open multiple times per day
- workflows that become habitual

Avoid:
- one-time utilities
- novelty features

---

### 5. Frictionless UX

- No signup required
- Instant functionality
- Minimal clicks to value

---

## 🧩 Feature Strategy

Prioritise features that:
- Increase frequency of use
- Reduce time to action
- Improve clarity

Avoid:
- unnecessary complexity
- rarely-used settings
- feature bloat

---

## 🎨 Design System

Use the **Calm Workspace** approach:

- Minimal, neutral UI
- Information-dense layout
- Fast scanning
- No visual noise

The product should feel like a **tool**, not an app.

---

## 📈 Growth Strategy

Think about distribution from day one:

- Chrome Web Store SEO (keywords in title + description)
- Clear screenshots showing value
- Simple demo/GIFs
- Shareable use-cases (X, Reddit, communities)

---

## 💰 Monetisation Paths

Design with monetisation in mind:

- Freemium (core free, advanced features paid)
- Power-user features
- Alerts, automation, or premium insights

---

## 🔁 Iteration Loop

1. Launch quickly
2. Gather feedback
3. Identify friction
4. Improve core experience
5. Repeat

Ship fast, refine continuously.

---

## 🧠 Mindset

- Think like a product builder, not a coder
- Build for users, not for yourself
- Optimise for usefulness, not complexity

---

## 🎯 End Goal

Become a **top Chrome Web Store developer** by building:

- simple
- useful
- high-retention tools

that people rely on daily.

---

## 🧠 Guiding Question

Before building anything, always ask:

> “Will someone use this every day?”

If not, rethink it.
